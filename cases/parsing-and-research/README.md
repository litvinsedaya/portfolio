# Кейс: парсинг и аналитика участников сообществ в соцсети

**Задача:** получение данных об участниках тематических сообществ определённой сферы, для последующего запуска платформы (анализ потенциальной аудитории: сотни групп, объём порядка **40 млн строк** с дубликатами по группам, десятки миллионов уникальных пользователей после дедупликации) для последующей сегментации и исследования.

---

## Часть 1. Парсинг данных

**Сложность:** API соцсети ограничивает частоту и однотипность запросов (Flood Control): лимиты запросов в секунду и срабатывания по повторяющимся паттернам. При объёме в десятки миллионов записей парсинг «в лоб» упирается в блокировки.

**Реализованные подходы:** ротация пула токенов с переключением при блокировке и прогрессивным временем «простоя»; учёт блокировок по методу API и токену; батчинг через `execute` (до 25 вызовов `users.get` в одном запросе); вставка «разнообразных» запросов (diversify) для снижения паттернности; throttling и экспоненциальный бэкофф при ошибке лимита; сохранение прогресса (state по группам/батчам) и выгрузка CSV по частям (~300k строк в файл) с возможностью возобновления после прерывания.

**Технологии и архитектура:** Python, библиотеки `requests`, `python-dotenv`. Вход: реестр групп (Excel). Выход: набор CSV-частей по группам, JSON state-файлы для resume; далее — склейка и дедупликация (Polars) в единый датасет (Parquet).

---

## Часть 2. Исследование и сегментация

**Подготовка данных:** сборка витрины на уровне пользователя из сырых CSV (дедупликация, нормализация полей, в т.ч. городов). Язык: Python. Библиотеки: **Polars** (основная работа с таблицами), **pandas** (чтение Excel, точечные операции). Итоговая витрина — партиционированные Parquet (например, `unique_users_part*.parquet`).

**EDA (разведывательный анализ):** анализ распределений по географии, активности, атрибутам профиля (в т.ч. свободные поля: occupation, activities, status, сайты, контакты); сводки по группам и категориям. Визуализация: **matplotlib** (batch-режим, сохранение в файлы). Данные: Polars, pandas, numpy.

**Кластеризация и сегментация:** сравнение методов (MiniBatchKMeans, Birch, GaussianMixture, AgglomerativeClustering) из **scikit-learn**; PCA и **StandardScaler** для препроцессинга; метрики (silhouette, Calinski–Harabasz, Davies–Bouldin). Финальная разметка сегментов — устойчивый метод (MiniBatchKMeans при выбранном k). Профилирование сегментов: описательные статистики и визуализации по кластерам (Polars, matplotlib).

**Выводы и маркетинг:** на основе EDA и сегментации сформулированы гипотезы о структуре аудитории; выполнена приоритизация сегментов (перспективность для привлечения); намечены маркетинговые стратегии: рекомендации по каналам (сообщества/группы по сегментам), офферам и месседжам, а также матрица идей для A/B-тестов. Результаты зафиксированы в отчёте (Markdown/HTML) с разделами по инсайтам и GTM-рекомендациям.
